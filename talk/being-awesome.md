# Talk Preparation Checklist: Being Awesome

**Purpose**: Verify the talk includes memorable "awesome" elements that strengthen technical understanding, not distract from it.  
**Created**: 2026-05-17  
**Source**: Being Awesome (mandatory) notes  
**Note**: Mark each item `[x]` only when the check is verifiably true.

## Core Requirement

- [ ] AWESOME001 Plan at least 2 "awesome" elements before slide production starts.
- [ ] AWESOME002 Confirm each "awesome" element directly supports a technical point.
- [ ] AWESOME003 Remove or revise any fun element that distracts from the technical story.
- [ ] AWESOME004 Verify the audience can remember the technical idea through the chosen element.

## Idea Development

- [ ] AWESOME005 During whiteboard planning, test at least 3 playful ideas (joke, metaphor, visual, interaction, or prop).
- [ ] AWESOME006 Keep at least 1 unconventional idea from brainstorming instead of defaulting to a safe version.
- [ ] AWESOME007 If a spontaneous metaphor or gesture appears in rehearsal, evaluate it for inclusion.

## Technique Options

- [ ] AWESOME008 Add at least 1 metaphor or analogy tied to a core concept.
- [ ] AWESOME009 Add at least 1 short story (fictional or true) that motivates the problem or solution.
- [ ] AWESOME010 Create at least 1 aphorism/catchphrase and reuse it across the talk.
- [ ] AWESOME011 Include at least 1 motion-based explanation (algorithm animation or staged reveal).
- [ ] AWESOME012 Evaluate whether 1 physical prop improves clarity for a key idea.
- [ ] AWESOME013 Evaluate whether visual media (photo/video/clip art/cartoon) improves recall for a key point.
- [ ] AWESOME014 Include at least 1 audience-participation moment (guess, vote, or show of hands) with reveal.
- [ ] AWESOME015 Add light wit at transition points without breaking flow.

## Performance and Delivery

- [ ] AWESOME016 Ensure the opening contains one attention-grabbing moment within the first 30 seconds.
- [ ] AWESOME017 Rehearse "awesome" moments out loud and confirm timing works naturally.
- [ ] AWESOME018 Confirm delivery remains clear if one "awesome" element fails (backup wording/slide path).
- [ ] AWESOME019 Keep energy high while preserving technical precision and pacing.

## Technical Alignment Check

- [ ] AWESOME020 Map each "awesome" element to one explicit technical learning outcome.
- [ ] AWESOME021 Verify no "awesome" moment requires background that the audience has not yet received.
- [ ] AWESOME022 Confirm key equations/claims/results are still understandable without relying only on humor.
- [ ] AWESOME023 In rehearsal, check whether listeners can restate both the fun hook and the technical message.

# Talk Preparation Checklist: Being Clear

**Purpose**: Verify the argument is structured as a story that listeners can follow and remember.  
**Created**: 2026-05-17  
**Source**: Being Clear notes  
**Note**: Mark each item `[x]` only when the check is verifiably true.

## Framing the Argument

- [ ] CLEAR001 Define the central claim in one sentence.
- [ ] CLEAR002 Define the narrative arc (start, tension, resolution) in 3-5 lines.
- [ ] CLEAR003 Verify every major point has a clear role in that arc.
- [ ] CLEAR004 Remove or rewrite any true point that does not fit the story.
- [ ] CLEAR005 Add one explicit "why this matters" statement near the start.
- [ ] CLEAR006 Confirm the audience can retell the story without needing all details.

## Story Patterns Across Fields

- [ ] CLEAR007 Add one lawyer-style framing line that ties facts into one case.
- [ ] CLEAR008 Add one politics-style framing line that prevents orphan points.
- [ ] CLEAR009 Add one math-style path from intuition to formal claim.
- [ ] CLEAR010 Add one business-style origin moment explaining why the idea exists.
- [ ] CLEAR011 Verify examples from other fields support clarity, not distraction.

## Choosing an Order

- [ ] CLEAR012 Write the idea graph (dependencies among concepts) before slide order.
- [ ] CLEAR013 Choose a linear sequence that preserves prerequisites for each step.
- [ ] CLEAR014 Place at least one motivating example before dense formal content.
- [ ] CLEAR015 For complex sections, include at least one intermediate stop (lemma, simpler case, or counterexample).
- [ ] CLEAR016 Add transition lines that explain why the next point follows now.
- [ ] CLEAR017 Confirm no step requires knowledge that appears later.

## "Oh, Just One More Thing" Flow

- [ ] CLEAR018 Structure at least 3 moments where the audience feels "complete" before one more valuable layer.
- [ ] CLEAR019 Ensure each added layer arrives within a few minutes and clearly rewards attention.
- [ ] CLEAR020 Use additions with concrete purpose (new problem, failure case, key insight, speedup, implementation detail, or full results).
- [ ] CLEAR021 Resolve likely audience concerns near where they arise instead of letting them fester.
- [ ] CLEAR022 Keep each increment small so people who lose the thread can still recover.
- [ ] CLEAR023 Include one explicit scope boundary (for example: deeper analysis deferred to paper).
- [ ] CLEAR024 Verify the talk still feels satisfying even if a listener disengages before the end.

## Pictures and Diagrams

- [ ] CLEAR025 Make most slides primarily pictorial rather than text-heavy.
- [ ] CLEAR026 Replace text-dense slides with diagrams or animations that convey the same idea.
- [ ] CLEAR027 For each abstract point, design at least one concrete visual example.
- [ ] CLEAR028 Verify each key idea has a drawable representation before finalizing the deck.

## Finding the Right Example

- [ ] CLEAR029 Introduce one concrete running example early in the talk.
- [ ] CLEAR030 Revisit the same example across multiple sections to show idea progression.
- [ ] CLEAR031 Ensure each major concept is applied to a familiar example before introducing a new one.
- [ ] CLEAR032 Verify the running example reduces cognitive load instead of adding side complexity.
- [ ] CLEAR033 If one example is insufficient, choose at most one additional complementary example.
- [ ] CLEAR034 Confirm the audience can connect later points back to the original example.

## Consistency

- [ ] CLEAR035 Choose a small fixed set of colors and fonts during storyboarding.
- [ ] CLEAR036 Assign stable meaning to each visual style (color, shape, emphasis) and keep it consistent.
- [ ] CLEAR037 Keep terminology and notation simple, and use the same terms throughout.
- [ ] CLEAR038 Reuse key graphic elements across slides instead of redrawing from scratch each time.
- [ ] CLEAR039 Verify no slide introduces conflicting style or vocabulary for an existing concept.

## Knowing What Not to Explain

- [ ] CLEAR040 Identify at least one topic that is too dense for the allotted time and cut it.
- [ ] CLEAR041 Replace cut deep dives with higher-level framing that preserves the story.
- [ ] CLEAR042 Verify every remaining section can be understood by non-experts at speaking pace.
- [ ] CLEAR043 Move out-of-scope details to backup slides, appendix, or paper reference.

## Opening Strong

- [ ] CLEAR044 Start with a real hook in the first 10-20 seconds (question, short story, surprising fact, or vivid scenario).
- [ ] CLEAR045 Add a clear transition that bridges the hook to the talk topic in one or two lines.
- [ ] CLEAR046 Give a brief self-introduction plus a short preview of what the talk will cover.
- [ ] CLEAR047 State the audience benefit explicitly: what they will be able to do or understand after this talk.
- [ ] CLEAR048 Remove "theme-song" filler (thanks/bio/setup) until after the audience is already hooked.

# Talk Preparation Checklist: Avoiding Common Mistakes

**Purpose**: Verify slides stay clear, readable, and easy to follow during live delivery.  
**Created**: 2026-05-17  
**Source**: Avoiding Common Mistakes notes  
**Note**: Mark each item `[x]` only when the check is verifiably true.

## Timing and Pace

- [ ] MISTAKE001 Target roughly 20 slides for a 20-minute talk as the baseline plan.
- [ ] MISTAKE002 If planning 30+ slides, keep each slide simple enough to explain quickly.
- [ ] MISTAKE003 Treat multi-slide animation sequences as one conceptual slide only when the audience perceives one evolving idea.
- [ ] MISTAKE004 Reserve time for audience decoding on each slide, not just speaking time.
- [ ] MISTAKE005 For bullet slides, budget explicit time to read or paraphrase each bullet out loud.
- [ ] MISTAKE006 For unfamiliar graphs, budget time to explain axes, legend, and what to look at.

## Layout and Density

- [ ] MISTAKE007 Use the full slide area instead of shrinking content into a small central region.
- [ ] MISTAKE008 Prefer plain backgrounds (white or black) when they improve usable layout space.
- [ ] MISTAKE009 Keep enough whitespace; do not fill every available area with content.
- [ ] MISTAKE010 Keep one main idea per slide unless side-by-side comparison is necessary.
- [ ] MISTAKE011 Split cluttered slides into multiple slides or staged builds.
- [ ] MISTAKE012 Remove cramped, cluttered, or wordy layouts before rehearsal.

## Text, Graphics, and Animation

- [ ] MISTAKE013 Keep bullet text short; replace long bullets with diagrams when possible.
- [ ] MISTAKE014 Confirm listeners can follow spoken explanation without needing to read dense text at the same time.
- [ ] MISTAKE015 Use animation mainly for reveal/hide steps that improve understanding.
- [ ] MISTAKE016 Use motion to show meaningful change (state evolution, graph improvement, information flow).
- [ ] MISTAKE017 Avoid decorative or busy animation that does not support comprehension.
- [ ] MISTAKE018 For formulas/tables/graphs, add clear labels so parts can be explained incrementally.

## Tool and Production Choices

- [ ] MISTAKE019 Confirm slide tooling supports deliberate layout and simple animation control.
- [ ] MISTAKE020 Avoid workflows that encourage bullet-heavy defaults when visual layout would teach better.
- [ ] MISTAKE021 If using TeX-heavy content, confirm formulas integrate cleanly in the chosen slide tool.

## Accessibility

- [ ] MISTAKE022 Use 24+ pt for most body text (28+ preferred when feasible).
- [ ] MISTAKE023 Limit ~20 pt text to secondary details (axis labels, annotations, dense references).
- [ ] MISTAKE024 Use readable sans-serif fonts consistently across slides.
- [ ] MISTAKE025 Include slide numbers so audience members can reference slides during Q&A.
- [ ] MISTAKE026 Ensure strong color contrast for readability on projectors and recordings.
- [ ] MISTAKE027 Avoid red-vs-green-only encoding when color carries semantic meaning.
- [ ] MISTAKE028 Keep a consistent visual language (fonts, colors, emphasis styles, notation).
- [ ] MISTAKE029 Ensure posted slides remain interpretable alone, or add notes/screencast context.
- [ ] MISTAKE030 Check assumptions against audience background diversity and preempt likely misunderstandings.

## Organization and Message Emphasis

- [ ] MISTAKE031 Highlight key words and key points visually and verbally.
- [ ] MISTAKE032 Mark whether points are good news, bad news, definitions, or surprises.
- [ ] MISTAKE033 Insert periodic section or "you are here" slides to maintain orientation.
- [ ] MISTAKE034 Use spoken signposting for structure ("tell them what you'll tell them...").
- [ ] MISTAKE035 Use strong, claim-based slide titles instead of vague titles.
- [ ] MISTAKE036 Consider a one-line takeaway message on each slide.
- [ ] MISTAKE037 Reconnect each section to the big picture, not just chronological project history.
- [ ] MISTAKE038 Explain principles and decision points that drove choices (not only "we did X, then Y").
